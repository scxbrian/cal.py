Building My First Website with Semantic HTML

Published on August 4, 2025 | By A Human Developer

About My Project

This page is my first attempt at building a website from scratch. My goal was to use only HTML5 and focus on using the right tags for the right job, rather than just using `div` everywhere. It's a bit of a meta project, reallyâ€”a web page about making a web page!

Key Takeaways from the Experience

  1. Semantic is not just for SEO!

    I learned that using tags like `header`, `nav`, and `main` makes the code so much more readable for me and other developers. It's like writing with proper grammar.

  2. Accessibility is easier than I thought.

    By simply using `alt` attributes on my images and a clear heading structure (`h2`, `h3`), I'm already making a huge difference for screen reader users. It's a little extra work, but it's totally worth it.

  3. Structure is everything.

    Before, I would just throw things on a page. Now, I see how a well-structured document, with `article` and `section` tags, creates a logical flow for the reader and a clear path for search engines.
